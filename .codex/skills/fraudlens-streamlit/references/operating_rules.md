# Reglas operativas - Streamlit

- Leer siempre `docs/arquitectura.md` y `docs/development.md` antes de editar.
- Clasificar cambios como `frontend`; agregar `seguridad` si se tocan textos de riesgo o advertencias.
- No calcular scoring complejo dentro de la UI.
- Mostrar que FraudLens genera alertas para revision humana.
- Registrar tarea completada, tarea pendiente y porcentaje de progreso.
