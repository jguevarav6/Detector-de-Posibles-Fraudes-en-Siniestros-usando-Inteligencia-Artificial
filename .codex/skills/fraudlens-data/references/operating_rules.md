# Reglas operativas - Data

- Leer siempre `docs/arquitectura.md` y `docs/development.md` antes de editar.
- Clasificar cambios como `datos` o `backend`.
- Usar solo datos sinteticos con IDs anonimos.
- No introducir nombres, correos, telefonos, documentos, placas reales ni credenciales.
- Registrar tarea completada, tarea pendiente y porcentaje de progreso.
