# Reglas operativas - Ethics

- Leer siempre `docs/arquitectura.md` y `docs/development.md` antes de editar.
- Clasificar cambios como `seguridad`, `documentacion` o `frontend` si se modifica UI.
- Usar lenguaje de alerta, posible riesgo y revision humana.
- Prohibir "fraude confirmado", "culpable", "rechazar automaticamente" y equivalentes.
- Registrar tarea completada, tarea pendiente y porcentaje de progreso.
